# CLAUDE.md — akk-portal

Публичный клиентский портал АО «Аграрная кредитная корпорация» (АКК). Этот репозиторий будет переименован `akk-railway` → `akk-portal`. Парный проект — `akk-office` (бывш. `akk-intelligence`): корпоративный сайт для сотрудников, живёт в отдельном репозитории, сюда не относится.

## Состояние и цель

- **Сейчас**: прототип = один [index.html](index.html) (~4435 строк: vanilla JS + CSS, без сборки) + рабочий Go-бэкенд в [backend/](backend/). Прод-демо: https://akk-production-4a6b.up.railway.app
- **Цель**: переписать фронт на Next.js в `web/`, перенести функционал agrocredit.kz (контентные разделы + воронка заявки), три языка (KZ/RU/EN), адаптив, деплой на Railway.
- **Главное правило**: бизнес-логика переносится **1-в-1, без додумывания**. Источники правды — [README.md](README.md) (раздел «Бизнес-логика») и сам `index.html`. Расчёты нового фронта должны совпадать с прототипом **до тенге**.

## Структура репозитория

| Путь | Что это |
| --- | --- |
| `index.html` | Легаси-прототип. **Не развивать** — только читать как эталон логики/дизайна. Удалим после паритета. |
| `__auth-integration.js` | Легаси-интеграция фронта с Go-бэком (SMS-auth, кабинет) — эталон для трека D. |
| `backend/` | Go/Echo + PostgreSQL: SMS-OTP, JWT, заявки, документы. **Рабочий, переиспользуем как есть.** |
| `web/` | Next.js-приложение — новый фронт. Вся новая разработка здесь. |
| `docs/PLAN.md` | План разработки: треки, задачи, зависимости, критерии приёмки. |
| `docs/STATUS.md` | Доска статуса задач. Обновляется каждой сессией. |
| `tests/` | Playwright E2E против легаси `index.html` (живы до конца миграции). |

## Рабочий процесс для агентов (обязательный)

1. Прочитай `docs/PLAN.md` и `docs/STATUS.md`.
2. Возьми назначенную задачу (или первую незаблокированную из своего трека). Отметь её в `STATUS.md` как `in_progress` до начала работы.
3. Выполняй строго по критериям приёмки из `PLAN.md`. Не выходи за рамки задачи.
4. Прогони тесты затронутой зоны. Обнови `STATUS.md` (`done` + 1 строка итога). Один коммит на задачу.
5. Параллельные сессии работают в разных треках — **не трогай файлы чужого трека** (карта владения — в `PLAN.md`).

## Команды

```bash
# web/ (Next.js — после задачи F1)
cd web && npm install
npm run dev          # дев-сервер
npm run build        # прод-сборка
npm run test         # vitest (unit)
npm run test:e2e     # Playwright

# backend/ (Go)
cd backend && go build ./...
go run ./cmd/server  # нужны env: DATABASE_URL, JWT_SECRET, DEMO_MODE=true для локалки

# легаси-прототип (пока жив)
npm run dev          # serve index.html на :4173
npm test             # Playwright против легаси
```

## Стек web/ (зафиксировано — не пересматривать без владельца проекта)

Next.js (App Router, **последняя стабильная** мажорная версия на момент скаффолда), TypeScript, Tailwind CSS, shadcn/ui, next-intl (локали `kk`/`ru`/`en` в роутинге), react-hook-form + zod, vitest (unit), Playwright (E2E). **Без БД и ORM на фронте** — все данные через Go-бэкенд (`AKK_BACKEND_URL` из env). Контентные страницы — SSG/SSR ради SEO.

## Инварианты бизнес-логики (нарушение = баг)

- `PROGRAMS` (7 программ), `scoreProgram`/`explainProgram`, `calculateSchedule` (схемы `biannual_winter` и `annual`), `FAJR_NORMS` (4 вида животных) — переносить дословно из `index.html` (≈ строки 2389–3053) и сверять с README.
- `rate` — число для калькулятора; `rateRange` — текст для отображения. Не схлопывать в одно поле.
- Программы с `indirectOnly: true` не показываются в результатах квиза — только в обзорной сетке.
- Порог попадания в результаты: `score >= 20`. Сортировка по убыванию score.
- Условные вопросы квиза: `purpose=livestock` → вопрос о виде животных; `animalType=КРС` → вопрос импорт/отечественный; прогресс-бар = реальное число шагов (5/6/7).
- Калькулятор — публичный pre-screen: **не** добавлять выбор ОС/ПОС/облигаций, **не** показывать полную форму Fajr (только Fajr-lite «хватит / не хватит»).
- Телефон: базовая маска `+7 (___) ___-__-__`, без жёсткой валидации.
- Единый стор воронки (ответы квиза + выбранная программа + калькулятор + стресс + форма заявки) — не разносить по разным сторам.
- События аналитики `track()` (имена + payload — таблица в README) сохранить полностью.

## Конвенции

- Коммиты, комментарии, документация — **на русском** (как вся история репозитория).
- UI-тексты — только через next-intl (`messages/{kk,ru,en}.json`). Русский — исходный; для kk/en при отсутствии перевода ставить русский текст с пометкой `// TODO перевод`, не выдумывать перевод.
- Дизайн-токены (эталон — `index.html:21–96`): `--primary #07663D`, `--accent #C9A21C`, `--bg #FAF6EC`; шрифты Montserrat (заголовки) / Onest (текст); тёмная тема обязательна; орнамент-паттерны из легаси.
- Адаптив до 360px, бургер-меню ≤640px, WCAG AA (focus-visible, skip-link, prefers-reduced-motion) — паритет с прототипом.
- **Никаких секретов в коде** — только env-переменные (Railway variables). В примерах — `<placeholder>`.
