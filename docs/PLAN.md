# План разработки akk-portal

Контекст и правила — в [CLAUDE.md](../CLAUDE.md). Статус задач — в [STATUS.md](STATUS.md).

Цель: фронт agrocredit.kz на Next.js (`web/`) + существующий Go-бэкенд (`backend/`), деплой на Railway. Бизнес-логика из `index.html` переносится 1-в-1.

## Треки и зависимости

```
Трек A (чистая логика, TS)  ──┐            может стартовать СРАЗУ, ни от чего не зависит
Трек F (фундамент web/)     ──┼─→ B, C, D  блокирует все UI-треки
                              │
F+A → Трек B (воронка UI)     │            параллельно с C и D
F   → Трек C (контент)        │            параллельно с B и D
F   → Трек D (auth+кабинет)   │            параллельно с B и C
F1  → Трек E (инфра/деплой)
```

**Параллельный запуск**: сессия 1 — трек A (сразу), сессия 2 — трек F (сразу). После F4: сессии на B, C, D одновременно. Владение файлами: A → `web/src/{data,lib}/`; B → `web/src/components/funnel/` + страница главной; C → `web/src/app/[locale]/(content)/` + `messages/`; D → `web/src/components/{auth,cabinet}/` + `web/src/lib/api/`; E → `Dockerfile`, конфиги. Общие файлы (layout, tailwind.config) меняет только трек F.

## Трек F — Фундамент (последовательно, одна сессия)

- **F1. Скаффолд Next.js в `web/`** — create-next-app (App Router, TS, Tailwind, ESLint), последняя **стабильная** версия; shadcn/ui init; vitest. Деплой-заготовка: порт из `process.env.PORT`.
  *Приёмка*: `npm run dev` и `npm run build` проходят; пустая главная рендерится.
- **F2. Дизайн-токены** — перенести CSS-переменные и тёмную тему из `index.html:21–96` в Tailwind-конфиг/`globals.css`; подключить шрифты Montserrat/Onest; орнамент-ассеты из легаси.
  *Приёмка*: страница-витрина `/dev/tokens` показывает палитру/типографику в светлой и тёмной теме.
- **F3. next-intl** — локали `kk`/`ru`/`en` в роутинге (`/[locale]/...`), `messages/{kk,ru,en}.json`, переключатель языка. Русский — исходный.
  *Приёмка*: главная открывается на `/ru`, `/kk`, `/en`; переключатель работает.
- **F4. Layout** — Header (лого АКК, навигация, тема-тоггл, слот авторизации, бургер ≤640px), Footer (3 колонки + орнамент), по образцу `index.html:2167–2204, 2325–2363`.
  *Приёмка*: адаптив до 360px; Playwright-скриншот мобильной шапки.

## Трек A — Бизнес-логика (чистые модули + unit-тесты; параллельно с F)

Все модули — чистый TypeScript без React, в `web/src/data/` и `web/src/lib/` (если F1 ещё не готов — писать в `web-logic/` и переместить). Каждая задача = модуль + vitest-тесты, цифры сверять с легаси.

- **A1. `data/programs.ts`** — типы + массив `PROGRAMS` (7 программ) из `index.html:≈2389–2505`, дословно: rate/rateRange/maxAmount/maxTerm/termByPurpose/scheduleType/hard/soft/featured/indirectOnly/hasStressTest. Плюс `PROGRAM_DETAILS` (`:≈2561–2657`).
- **A2. `data/questions.ts`** — `QUESTIONS` (5 базовых) + `ANIMAL_TYPE_Q`/`CATTLE_ORIGIN_Q`/`HEADS_Q` + логика `getQuestions()` (условные ветки, 5/6/7 шагов) из `:≈2659–2746`.
- **A3. `data/fajr-norms.ts`** — `FAJR_NORMS` по 4 видам животных из `:≈2751–2811`, без изменений коэффициентов.
- **A4. `lib/scoring.ts`** — `scoreProgram`/`explainProgram`/`scoredPrograms` из `:≈2940–2992` (hard-фильтры AND, soft-веса, `min(100, Σw)`, порог `>=20`, разбор недобора «до +N»).
  *Приёмка*: тесты на 7 сквозных сценариев из `tests/onboarding.spec.js` (ВПРиУР, импортный скот 100–499 / 500+, отечественный, микрокредит, откорм, инвестиции) — топ-программа и % совпадают с легаси.
- **A5. `lib/schedule.ts`** — `calculateSchedule` (biannual_winter: 50/50 на 5 дек / 5 мар, проценты 8+3 мес; annual: линейное тело + проценты на остаток) + `effectiveMaxTerm`/`pickInitialTerm` из `:≈3008–3053`.
  *Приёмка*: тесты — Кең дала 2 (двухплатёжный), Игілік 6%/84 мес, Агробизнес с целью ПОС → срок ≤48 мес. Суммы до тенге как в легаси.
- **A6. `lib/stress.ts`** — `calculateStress` (доход молоко+мясо+доп, расходы × поголовье, ratio платёж/чистый доход; проверки пастбищ/помещений; вердикты <30/30–50/>50%) из `:≈4118–4250` (расчётная часть).
- **A7. `lib/format.ts`** — `fmtAmount`, `fmtRate`, `declension`.

## Трек B — Воронка заявки UI (после F4 + A; одна сессия)

- **B1. ProgramGrid + ProgramModal** — сетка 6 программ (Кең дала скрыта: `indirectOnly` — бейдж «только через КТ/МФО/БВУ»), модалка с деталями из `PROGRAM_DETAILS`, кнопка «Подать заявку».
- **B2. Quiz** — шаги с прогрессом, условные вопросы, кнопка «Назад», переход по пройденным шагам; единый стор воронки.
- **B3. Results** — карточки с % совпадения, бейдж «Лучшее совпадение», раскрывашка «Почему N%» (`explainProgram`, недоборы «до +N»).
- **B4. Calculator** — слайдер + ручной ввод суммы, кнопки сроков (фильтр по `effectiveMaxTerm`), таблица графика (две схемы), платёж/переплата.
- **B5. StressTest** — форма (вид животных, поголовья, пастбища, помещения, платежи, доход) + вердикт-пилюля и разбор. Показывать только для `hasStressTest` программ, с кнопкой «Пропустить».
- **B6. ApplyWizard + Success** — банковский визард подачи (эталон — легаси-коммит `e623659` и далее), экран успеха с номером заявки.
- **B7. Главная** — Hero (эмблема, статистика, CTA) + сборка секций; событийная аналитика `track()` на все шаги (имена/payload — README, таблица «Аналитика и events»).
  *Приёмка трека*: все 7 квиз-сценариев проходят E2E; цифры калькулятора/стресса = легаси.

## Трек C — Контентные страницы (после F3/F4; одна сессия, можно дробить)

- **C1. Шаблон контентной страницы** — SSG, i18n, SEO-мета (title/description/og), хлебные крошки. Дальше — заполнение по разделам agrocredit.kz:
- **C2.** О компании: стратегия/миссия, структура (СД, правление, дочерние), корпоративное управление, комплаенс, омбудсмен.
- **C3.** Отчётность (финансовая/годовая) + «Инвесторам» + рейтинги — списки документов (PDF-ссылки).
- **C4.** Пресс-центр: новости + СМИ о нас + истории успеха (локальный контент/MDX; CMS — отдельная итерация).
- **C5.** FAQ (аккордеон).
- **C6.** Контакты: центральный аппарат + 16 филиалов + кредитные партнёры (карточки по областям).
- **C7.** Агрострахование, продажа залогового имущества, закупки, вакансии.
  *Приёмка трека*: каждая страница доступна на 3 локалях, текст в server-rendered HTML (view-source), нет хардкода строк вне `messages/`.

## Трек D — Auth + личный кабинет (после F4; одна сессия)

Бэкенд готов — эндпоинты в `backend/cmd/server/main.go`; эталон клиентской логики — `__auth-integration.js`.

- **D1. API-клиент** — `web/src/lib/api/` поверх `AKK_BACKEND_URL`: auth (`RequestSms`, `VerifySmsCode`, `smsRegister`, `CheckBmgAndSendSmsForRegister`, `ssoDemoLogin`, `me`) + credit (`applications` CRUD, `advance`, `documents`); хранение токенов, разбор JWT-клеймов.
- **D2. SMS-auth UI** — логин по ИИН → OTP (6 ячеек), регистрация (ФИО+ИИН+телефон → OTP), demo-код в demo-режиме; модалка по образцу легаси.
- **D3. Кабинет: список заявок** — статус-пилюли, сводка, пустое состояние, CTA «Подобрать программу».
- **D4. Страница заявки** — трекер 7 этапов + ветка отказа, каталог требований к документам, отметка загрузки.
- **D5. SSO demo** — кнопки eGov/Baiterek (официальные логотипы из легаси), demo-флоу.
  *Приёмка трека*: с локальным Go-бэком (`DEMO_MODE=true`) проходит полный путь: регистрация → подбор → заявка → кабинет → движение статуса → документы.

## Трек E — Инфра и деплой

- **E1. Dockerfile + Railway** (после F1) — multi-stage Node-образ для `web/`, `PORT` из env, `AKK_BACKEND_URL` через Railway variables; сервис рядом с существующим Go-сервисом.
- **E2. Перенос Playwright E2E** (после B) — адаптировать `tests/onboarding.spec.js` + мобильный скриншот-тест к новому фронту; селекторы по текстам, не по классам.
- **E3. Аналитика** (после B7) — `track()` → консоль-коллектор сейчас, интерфейс под GA4/Метрику.
- **E4. Переименование репо** `akk-railway → akk-portal` (GitHub Settings → Rename, обновить remote; Railway-привязку перепроверить). **Только после подтверждения владельца.**
- **E5. Переключение прод-домена** на новый фронт; легаси `index.html` в `legacy/` или удалить. **Финал, после паритета и приёмки.**

## Что НЕ делать (из README, обязательное)

- Не упрощать `rate` до одного числа при наличии диапазона.
- `indirectOnly`-программы — не в результаты квиза.
- Не превращать калькулятор в форму с ОС/ПОС/облигациями; не показывать полный Fajr.
- Не валидировать телефон жёстко.
- Не редактировать `index.html` (заморожен как эталон).
