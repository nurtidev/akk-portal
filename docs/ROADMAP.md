# Карта развития сайта akk-portal

Верхнеуровневая карта: что уже в проде, что на паузе и куда движемся. Это
обзор для владельца и команды; детальные задачи и критерии — в [PLAN.md](PLAN.md),
факт выполнения — в [STATUS.md](STATUS.md). Этот файл показывает **направление и
зависимости**, а не статусы по каждой задаче.

Обновлено: 2026-06-29.

---

## 1. Где мы сейчас (в проде / `main`)

Публичный портал на Next.js (`web/`) переписан с легаси-прототипа, задеплоен на
Railway. Готово и работает:

- **Лендинг**: кино-герой, секция «Под разные цели АПК» (выбор программы стрелками
  + калькулятор платежа с раскрывающимся графиком), обзор программ, «Как получить
  финансирование», обращение председателя, пресс-центр, устойчивое развитие.
- **Воронка подбора**: квиз (5–7 шагов) → результаты по совпадению → калькулятор →
  стресс-тест (для Игілік/Береке) → подача заявки. Единый стор, бизнес-логика 1-в-1.
- **Отдельная страница `/calculator`** (публичный pre-screen).
- **Контентные разделы**: о компании, отчётность, инвесторам, пресс-центр, FAQ,
  контакты/филиалы, как получить, проблемная задолженность, инвестпроекты, блог,
  устойчивое развитие, страхование/залоги/закупки/вакансии, Конституция РК.
- **Страницы продуктов** `/products/<id>` + брошюры программ (4 программы).
- **Кабинет клиента**: вход по SMS-OTP и SSO (eGov/Baiterek, демо), список заявок,
  трекер этапов, документы.
- **Три языка** (ru/kk/en), тёмная тема, адаптив, WCAG-доработки.

## 3. Куда движемся

### Блок А. АгроСкор: из демо в функционал ⭐ (разблокирует «Партнёрам»)

Сейчас движок скоринга (`web/src/lib/agroscore.ts`), раздел в кабинете и витрина
«Банкам-партнёрам» работают на **мок-данных / демо-персонах**. Чтобы открыть
«Партнёрам» и показывать клиенту реальный балл:

- определить источник данных скоринга (бэкенд: 1С / OKAPS / БВУ / гос-реестры);
- реальный расчёт балла на бэке вместо фронтового мока;
- согласовать правовую рамку передачи данных банкам;
- после готовности — вернуть пункт «Партнёрам» в навигацию.

### Блок Б. Бэкенд-интеграции воронки (сейчас мок на фронте)

- **Реальная отправка SMS** (G10) — код OTP сейчас генерится на фронте (демо).
- **Публичный `POST /leads`** без авторизации (G11) — заявка гостя/консультация
  сейчас даёт синтетический номер.
- **`POST /api/blog/question`** (G12) — форма вопроса председателю шлёт в заглушку.

### Блок В. Контент и данные от владельца (блокеры — нужны материалы)

- Реальные ссылки **WhatsApp/Telegram** в футере (G5).
- **Адреса филиалов** (G7) — сейчас плейсхолдер «уточните по 1408».
- **Ссылки раскрытия для инвесторов** (G8) — кнопки скрыты до реальных URL.
- **Картинки владельца** (G9) — лого банков, фото программ (есть фолбэки;
  промты в [IMAGE_PROMPTS.md](IMAGE_PROMPTS.md)).
- **Решения по брошюрам** — см. [BROCHURES.md](BROCHURES.md): 3 листовки без
  программы (Жайлау / ПОС-животноводство / Переработка), расширенный охват

## 5. Принципы (не нарушать)

- Бизнес-логика переносится 1-в-1; расчёты совпадают с прототипом до тенге.
- Бизнес-данные программ (`programs.ts`, веса скоринга, ставки) меняются только
  по подтверждению владельца.
- Никаких секретов в коде — только env / Railway variables.
- UI-тексты — через next-intl; казахский/английский без выдуманных переводов.
